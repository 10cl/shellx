## Authentication

All API endpoints require authentication using API keys or Bearer tokens. You can obtain your API key from the ShellX.ai dashboard.

### API Key Authentication

```bash
curl -H "Authorization: Bearer YOUR_API_KEY" \
  https://api.shellx.ai/v1/terminal/sessions
```

### JWT Token Authentication

```bash
curl -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  https://api.shellx.ai/v1/ai/process
```

## Base URL

All API endpoints are relative to the base URL:

```
https://api.shellx.ai/v1
```

## Rate Limiting

API requests are rate-limited to ensure fair usage:

- **Free Tier**: 100 requests per hour
- **Pro Tier**: 1,000 requests per hour
- **Enterprise**: Custom limits

When rate limits are exceeded, the API returns a `429 Too Many Requests` status code.

## Error Handling

ShellX.ai API uses standard HTTP status codes and returns detailed error messages:

```json
{
  "error": {
    "code": "INVALID_COMMAND",
    "message": "The specified command could not be processed",
    "details": "Command syntax error at position 15"
  }
}
```

## Quick Start Example

Here's a simple example of using the ShellX.ai API:

```javascript
const ShellX = require('@shellx/sdk');

const client = new ShellX({
  apiKey: 'your-api-key-here'
});

// Process natural language command
const result = await client.ai.process({
  input: "show me all files in the current directory",
  context: {
    workingDirectory: "/home/user/projects"
  }
});

console.log(result.command); // "ls -la"
console.log(result.output);  // File listing
```

ShellX.ai supports webhooks for real-time notifications:

```json
{
  "url": "https://your-app.com/webhook",
  "events": ["command.executed", "error.occurred", "session.ended"],
  "secret": "your-webhook-secret"
}
```

## API Versioning

The API uses semantic versioning. Current version is `v1`. Breaking changes will result in a new version number.
