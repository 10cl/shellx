## Configuration Overview

ShellX.ai provides extensive configuration options to customize your AI-powered terminal experience. Settings can be configured through the web interface, configuration files, or command-line options.

## Global Configuration

### Basic Settings

Configure basic ShellX.ai behavior:

```json
{
  "ai": {
    "model": "gpt-3.5-turbo",
    "temperature": 0.7,
    "maxTokens": 1000,
    "timeout": 30
  },
  "terminal": {
    "theme": "dark",
    "fontSize": 14,
    "fontFamily": "Monaco, monospace",
    "autoComplete": true
  },
  "security": {
    "apiKey": "your-api-key",
    "maxSessions": 5,
    "sessionTimeout": 3600
  }
}
```

### AI Model Configuration

Choose and configure AI models:

```javascript
// Available models
const models = {
  "gpt-3.5-turbo": {
    name: "GPT-3.5 Turbo",

  },

    name: "Claude-3",

};
```

## Terminal Customization

### Theme Configuration

Customize the terminal appearance:

```css
/* Dark Theme */
.shellx-terminal.dark {
  background-color: #1a1a1a;
  color: #ffffff;
  border-color: #333333;
}

/* Light Theme */
.shellx-terminal.light {
  background-color: #ffffff;
  color: #000000;
  border-color: #cccccc;
}

/* Custom Theme */
.shellx-terminal.custom {
  background-color: #2d2d2d;
  color: #f8f8f2;
  border-color: #44475a;
}
```

### Font and Display

Configure text rendering:

```json
{
  "display": {
    "fontFamily": "JetBrains Mono, Monaco, monospace",
    "fontSize": 14,
    "lineHeight": 1.5,
    "letterSpacing": 0.5,
    "cursorStyle": "block",
    "cursorBlink": true
  }
}
```

## AI Assistant Settings

### Response Preferences

Configure how the AI responds:

```json
{
  "ai": {
    "responseStyle": "concise", // "concise", "detailed", "educational"
    "showExplanations": true,
    "suggestAlternatives": true,
    "learnFromFeedback": true,
    "contextWindow": 10 // Number of previous commands to remember
  }
}
```

### Command Processing

Fine-tune command interpretation:

```json
{
  "commandProcessing": {
    "confirmDestructive": true,
    "autoCorrect": true,
    "suggestSimilar": true,
    "validateSyntax": true,
    "timeoutSeconds": 30
  }
}
```

## Security Settings

### Authentication

Configure authentication methods:

```json
{
  "authentication": {
    "method": "jwt", // "jwt", "apikey", "oauth"
    "tokenExpiry": 3600,
    "refreshTokens": true,
    "mfa": {
      "enabled": true,
      "methods": ["totp", "sms"]
    }
  }
}
```

### Access Control

Set up access controls:

```json
{
  "accessControl": {
    "allowedCommands": ["ls", "cat", "grep", "find"],
    "blockedCommands": ["rm", "sudo", "chmod"],
    "maxConcurrentSessions": 3,
    "ipWhitelist": ["192.168.1.0/24"],
    "rateLimiting": {
      "enabled": true,
      "maxRequests": 100,
      "windowMinutes": 60
    }
  }
}
```

## Configuration Management

### Configuration Files

ShellX.ai looks for configuration files in the following order:

1. `~/.shellx/config.json` (User-specific)
2. `/etc/shellx/config.json` (System-wide)
3. `./shellx.config.json` (Project-specific)

### Environment Variables

Override configuration with environment variables:

```bash
export SHELLX_AI_MODEL=gpt-4
export SHELLX_API_KEY=your-api-key
export SHELLX_THEME=dark
export SHELLX_DEBUG=true
```

### Command Line Options

Override settings via command line:

```bash
shellx --model gpt-4 --theme light --debug
```

## Validation and Testing

### Configuration Validation

Validate your configuration:

```bash
# Validate configuration file
shellx config validate

# Test API connectivity
shellx config test-api

# Check AI model availability
shellx config test-ai
```

### Configuration Templates

Use predefined configuration templates:

```bash
# Initialize with default template
shellx config init --template default

# Use development template
shellx config init --template development

# Use production template
shellx config init --template production
```

<Note>
  Configuration changes require a restart of the ShellX.ai service to take effect.
</Note>

<Tip>
  Use `shellx config --help` to see all available configuration commands and options.
</Tip>
